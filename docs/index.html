Abstract

Predicting how air pressure distributes itself over a wing is one of the central problems in aerodynamic design. This distribution—the pressure coefficient (Cp) at every point along an airfoil's surface—determines critical aerodynamic performance indicators such as lift, drag, and structural loads.

Typically, Cp curves are obtained by running expensive Reynolds-Averaged Navier-Stokes (RANS) CFD simulations. Since each simulation can take from several minutes to hours, exploring the vast multi-dimensional operating space during early-stage aircraft design is computationally prohibitive.

The SHARP-Raptor Challenge addresses this bottleneck as a high-speed machine learning benchmark for surrogate modeling. Given only two inputs—the freestream Mach number and the angle of attack (α)—your task is to build a model capable of accurately predicting the complete Cp distribution across 13,862 physical points of an F-22 fighter airfoil section.

By substituting expensive physics solvers with instantaneous ML inference, this challenge aims to evaluate the limits of ML models in learning and generalization across highly non-linear subsonic, transonic, and supersonic flow regimes.

Database

All samples in this dataset correspond to the exact same fixed airfoil geometry (an F-22 Raptor airfoil section), evaluated across various flow conditions. The geometry and point distribution remain identical across all cases; only the flow environment changes.

Each flow condition is parameterized by:

  • Mach number (mach): Freestream Mach number, ranging from 0.3 to 1.4 (covering subsonic, transonic, and supersonic regimes).
  • Angle of attack (alpha / aoa): Angle of attack in degrees, ranging from 0° to 5°.

For each combination of Mach and α, the pressure coefficient (Cp) is provided at 13,862 fixed points along the airfoil's boundary. The same physical points (mapped via point_id) are preserved across all data files. Note: The Reynolds number is held constant for all cases and is not required as an input.

Challenge Files

The dataset consists of the following files, which can be easily linked together using the point_id or id variables:

1. train.csv

The training set, containing 85 complete cases (85 conditions × 13,862 points = 1,178,270 rows). The target label (cp) is fully provided.

Column Type Description
id string Unique row identifier
case_id int Flow-condition identifier (0–70)
point_id int Surface point identifier (0–13861), consistent across all files
mach float Freestream Mach number (0.3 to 1.4)
aoa float Angle of attack in degrees (0 to 5)
cp float Target — pressure coefficient at the given point
2. test.csv

The evaluation set, containing 15 unvalidated flow conditions (15 conditions × 13,862 points = 207,930 rows). This file is identical to train.csv, but the cp column is withheld.

3. airfoil_coordinates.csv

The spatial coordinate map of the fixed F-22 section geometry. You can join this table with train/test sets using point_id to incorporate physical coordinates into your spatial models.

Column Type Description
point_id int Surface point identifier
x float Chordwise spatial coordinate
y float Surface-normal spatial coordinate
4. validation_cases.csv

Contains the exact flow condition IDs used in our validation splits for hyperparameter tuning. It is provided for replicability purposes.

5. sample_submission.csv

An example submission template showing the exact format required. It includes the id column and a cp prediction column (pre-filled with zeros).
